# Submission Checklist

Use this before submitting on BlockseBlock.

## Mandatory Links

- [ ] Deployed application link is public and working.
- [ ] GitHub repository link is public or accessible to judges.
- [ ] Google Doc project description link is accessible to anyone with the link.

## Google Doc Must Include

- [ ] Problem Statement Selected: PS1 - The Last-Minute Life Saver
- [ ] Solution Overview
- [ ] Key Features
- [ ] Technologies Used
- [ ] Google Technologies Utilized

## Functional Checks

- [ ] App loads at deployed URL.
- [ ] Situation intake works.
- [ ] Deadline parsing works for `hours`, `minutes`, and `at 5:30pm`.
- [ ] Gemini API key is configured in deployment environment as `VITE_GEMINI_API_KEY`.
- [ ] Triage cards appear after activation.
- [ ] Rescue plan appears.
- [ ] Check-in button works.
- [ ] Mark done button works.
- [ ] App remains usable if Gemini returns a rate-limit error.
- [ ] Mobile layout works.

## BlockseBlock Steps

1. Go to `https://blockseblock.com/dashboard`.
2. Click the hackathon project area.
3. Click `Create Project`.
4. Enter project name: `LimitBreaker AI`.
5. Select problem statement: `PS1 - The Last-Minute Life Saver`.
6. Add deployed application link.
7. Add GitHub repository link.
8. Add Google Doc project description link.
9. Toggle both confirmation notes.
10. Click `Final Submit` only after all links are verified.
